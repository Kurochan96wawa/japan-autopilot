# 自動運用の落とし穴 ＆ 本ツールの対策（対応表）

ネット上で繰り返し報告される「SNS自動運用・Pinterest・アフィリ・AI記事・API運用」の
典型的な失敗を集め、各々にこのツールがどう対処しているかをまとめた。

> 🔴 大前提（正直に）：**「完全無欠」は存在しない**。プラットフォームの規約は予告なく変わり、
> BAN対策は“確率を下げる”もので“ゼロにする”ものではない。本ツールは
> **既知の落とし穴を体系的に潰し、事故率を実運用レベルまで下げる**ことを目的にしている。
> 残る最後の保険は「月10分の目視」と「攻めすぎないこと」。

---

## A. プラットフォームBAN / shadowban 系

| 落とし穴 | 本ツールの対策 |
|---|---|
| 投稿が多すぎ/速すぎ（Pinterestは1日5〜15・1時間25〜30が目安） | `safety.max_pins_per_day_hard`(既定12)で**絶対上限**。超過分は自動停止 |
| 新規アカの初動でいきなり大量投稿→即凍結 | **ウォームアップ**：最初の`warmup_weeks`(3週)は1日`warmup_pins_per_day`(3)に自動制限 |
| 機械的な等間隔投稿でbotだとバレる | 投稿間隔に**ジッター**（`jitter_pct`±40%）を入れて人間らしく |
| 同一/類似**画像**の連投（最大のshadowban要因） | Fresh Pins：再Pinは毎回別写真+別帯デザイン。**perceptual hashで類似画像を検出して弾く** |
| 同一**説明文**の使い回し | Jaccard類似で**酷似テキストを検出して弾く** |
| 同一**URL**への連投 | `min_days_between_repins`(既定7日)で間隔を強制。同日再Pinは不可 |
| 非公式の自動化ツールが検出される | **公式API（Pinterest v5 / Meta Graph）のみ**使用。スクレイピングや非公式SDKは不使用 |
| いいね/フォロー/コメントの自動化（最も厳しく罰せられる） | **やらない**。本ツールは“コンテンツ作成と投稿”だけを自動化。エンゲージメントは人間（=あなた）が行う |
| shadowbanに気づかず投稿し続けて悪化 | 週次でインプレッションを監視し、急落(ベースライン比30%未満)を検知すると**自動で投稿停止**(`pause_on_shadowban`) |
| 2:3比率でない/キーワード無し説明 | 画像は1000×1500(2:3)固定、説明はキーワード入りで生成 |

## B. AI記事 / Google低品質判定 系

| 落とし穴 | 本ツールの対策 |
|---|---|
| 薄いAI記事の量産（Scaled content abuseで50〜80%流入減の実例） | **品質ゲート**：`min_article_words`(450)未満や見出し無しは破棄。AI自己採点で薄ければ1回作り直し |
| どのページも同じ構造でオリジナリティ皆無 | ニッチ特化のpersona/positioningで切り口を変え、ロングテール特化。再Pinも“別アングル”で生成 |
| 一気に数百ページ公開→量産判定 | 1日数件のウォームアップ運用。**速度で殴らない**設計 |
| 著者情報なし（E-E-A-T欠如） | 記事に開示・出典(Pexels)を明記。※さらに強くするなら著者/Aboutページ追加を推奨（CAVEATS参照） |
| 古くなる情報（価格・営業時間）で誤情報化 | **リスク表現フィルタ**：具体価格・営業時間・誇大表現を含む記事は破棄(`block_risky_phrases`) |

## C. リンク / 導線 系

| 落とし穴 | 本ツールの対策 |
|---|---|
| アフィリ直リンクの大量投下でスパム判定 | Pinは**自前のHub(GitHub Pages)経由**。直リンクしない |
| リダイレクト/短縮URL/クローキングが弾かれる | 生のアフィリURLをHub記事内に設置。短縮・クローキング不使用 |
| 死んだリンク/薄い飛び先＝赤信号 | アフィリURLの**死活・placeholder判定**。未設定/死リンクは記事に入れない(`check_affiliate_links`) |
| 開示なしでFTC罰金（最大$53,088/件）/Amazon垢閉鎖 | 記事フッターに開示文、アフィリ誘導Pinに**`#ad #affiliate`を自動付与** |

## D. アフィリプログラム 系

| 落とし穴 | 本ツールの対策 |
|---|---|
| Amazon：180日で3件売れないと垢閉鎖→再申請困難 | 既定で全国版より転換しやすいKlook/Booking等を主力に。**Amazonは流入が出てから申請**するよう明記(affiliates.yaml) |
| Amazon：独自開示文が必須 | "As an Amazon Associate…"を開示文に同梱済み |
| JRパス全国版の罠（値上げで「元取れない」認識） | 全国版を外し地域パス・体験ツアーに差し替え済み |

## E. インフラ / API 系

| 落とし穴 | 本ツールの対策 |
|---|---|
| Metaトークンが60日で失効→再認証地獄 | `tokens.py`+`refresh-tokens.yml`で**20日ごとに自動更新**し、新トークンをSecretsへ書き戻し |
| Pinterestトークンも短命(約30日) | 同上。refresh_tokenで自動更新 |
| GitHub Actions cronが遅延/スキップ（±1h・混雑時欠落） | 時刻厳守に依存しない設計。`workflow_dispatch`で手動実行も可 |
| 60日コミット無しでスケジュール**自動無効化**（サイレント） | 毎日state/siteを**commit**するので自然に回避 |
| 失敗してもサイレント故障で気づかない | 失敗時に**GitHub Issueを自動起票**して通知（daily/refresh両方） |
| LLM/APIコストの暴走 | `monthly_llm_call_cap`(400)で上限。超えたら当月は生成停止 |
| 機密のハードコード漏洩 | 全鍵はGitHub Secrets/.env。`.gitignore`済み。コードに鍵を書かない |

## F. 倫理 / 持続性 系

| 落とし穴 | 本ツールの対策 |
|---|---|
| 偽ペルソナ/なりすまし | 実在人物を騙らない。価値ある情報提供に徹する方針をeditorial_rulesで強制 |
| 量産スパムで世の中のノイズを増やす | 「世のため人のため」を編集方針に明記。薄い記事は品質ゲートで自動破棄 |

---

## それでも残る“最後のリスク”（隠さない）
1. **規約変更**：各社の方針は変わる。半年に一度はCAVEATS/PITFALLSを見直す前提。
2. **写真の独自性**：ストック写真だけでは実体験勢に視覚で劣る。自分の写真を足す半自動が最強。
3. **収益の立ち上がり**：依然2〜4ヶ月は仕込み。即金ではない。
4. **完全0タッチの過信**：月10分の目視が最良の保険。これだけは自動化しない方が安全。
